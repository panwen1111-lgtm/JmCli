# 豆沙算力云端部署指南 (Phase 2)

本指南旨在帮助你实现 **“云端控制台 + 本地算力节点”** 的分布式架构。

## 1. 分解清单 (哪些文件去哪里)

### 🚀 云端服务器 (Central Server)
云服务器只需要处理任务状态和网页展示，**不需要** `dreamina.exe`。
- `server.js` (中控逻辑)
- `database.js` (数据持久化)
- `cli_adapter.js` (虽然不运行 exe，但 server 启动时需要加载该类定义)
- `public/` (所有前端 HTML 页面)
- `package.json` (依赖定义)

### ⚙️ 本地算力端 (Local Worker)
你本地需要显卡和账号环境的机器。
- `worker.js` (执行节点)
- `cli_adapter.js` (执行逻辑)
- `../dreamina.exe` (官方 CLI)

---

## 2. 部署步骤

### 第一步：服务器配置 (云端)
1. 将服务器文件上传至服务器目录。
2. 安装环境：`npm install`。
3. 启动服务：`node server.js`。
4. **重要**：在云服务器控制台（安全组）中开放 **3000** 端口。

### 第二步：供应商托管 (网页)
1. 在浏览器访问：`http://你的云服务器IP:3000/provider`。
2. 完成账号托管（自动或手动回填 JSON）。

### 第三步：启动算力 (本地)
1. 在本地打开命令行，进入 `dousha_network_mvp` 目录。
2. 运行指令：
   ```bash
   node worker.js http://你的云服务器IP:3000
   ```
3. 你会看到提示：`[Worker] 正在连接服务器: http://你的云服务器IP:3000`。

---

## 3. 安全建议
- **不要公开 URL**：目前的 MVP 版本未加入登录密码，建议不要将 URL 传播给他人。
- **端口修改**：如果 3000 端口被占用，可以在 `server.js` 顶部的 `PORT` 变量修改。
