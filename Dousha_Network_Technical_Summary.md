# 豆沙云端算力共享平台 (Dousha Network) 技术总结报告

## 1. 项目背景与目标
针对“豆沙漫剧助手”从本地向云端转型的需求，本项目旨在构建一个**中心化调度 + 分布式执行**的算力共享平台。
- **核心目标**：实现“账号一处登录，云端随处可用”，并能通过共享闲置算力（积分）进行价值交换。

## 2. MVP 方案概览 (CLI 模式)
在经过多轮尝试后，我们最终弃用了不稳定的 Puppeteer UI 模拟方案，转向了更稳健的 **CLI (`dreamina.exe`) 调用方案**。

### 核心实现链路：
1.  **Session 捕获**：用户通过 `dreamina login --debug` 获取登录后的响应 JSON。
2.  **任务分发**：中控服务器 (Server) 接收任务和对应的 Session JSON。
3.  **动态注入**：执行节点 (Worker) 领取任务后，通过 `import_login_response` 将 Session 注入到本地环境。
4.  **自动执行**：调用 `text2image` 指令和相应的轮询机制。

## 3. 实现要点 (Implementation Key Points)
- **Session 脱敏平移**：成功的关键在于利用 CLI 的 `import` 机制，实现 Worker 节点的“无状态化”。
- **异步任务观察**：支持 `--poll=1` 机制，使 Node.js 同步观察即梦排队状态。
- **分布式潜力**：当前架构支持横向扩展，Worker 节点可以分布在任意 Windows Server 环境。

## 4. 可行性验证结论 (Feasibility Confirmed)
> [!NOTE]
> **验证成功**：通过 `test_submit.js` 提交任务后，Worker 成功识别登录态并触发了后端生成流程。标志着“云端算力共享”的闭环已经走通。

## 5. 后续演进建议 (Roadmap)
既然 MVP 核心逻辑已验证通过，你可以考虑以下关键演进方向：
- **接入持久化**：把 Session 存储到 Redis 或数据库中，确保服务重启后托管状态不丢失。
- **多 Worker 并发**：测试多台执行机同时领任务，验证分布式算力池的扩展能力。
- **前端中控台**：做一个网页版管理端，让用户可以直接通过 UI 粘贴 JSON 进行托管并监控任务。

### 更多详细规划建议：
#### 短期 (工程化升级)
- **持久化队列**：引入 Redis (BullMQ) 提升吞吐量。
- **环境隔离**：通过环境变量隔离 Worker 实例，避免 Session 文件冲突。

### 中期 (业务化加固)
- **Session 自动刷新**：研究长效维持机制，降低用户扫码频率。
- **多模态扩展**：接入 `image2video` 和 `multimodal2video` 等全能指令。

### 长期 (平台化建设)
- **积分撮合引擎**：自动匹配高等级会员账号优先执行。
- **Web UI 仪表盘**：可视化监控算力池状态。
